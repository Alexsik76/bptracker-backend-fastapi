# Промпт для агента — поштова інфраструктура (крок 1 з 2 для magic-link)

## Мета

Створити мінімальну поштову підсистему: асинхронний відправник на `aiosmtplib` за
стабільним інтерфейсом `EmailSender`. Це спільна інфраструктура, а **не** доменний модуль;
живе поза модулями, поряд із конфігурацією та підключенням до БД. Логіку magic-link у цьому
кроці **не** чіпаємо — тут лише транспорт.

## Незмінні правила проєкту

- **Мова.** Увесь код, рядкові літерали конфігурації та коментарі — виключно англійською.
  Жодного українського тексту у файлах коду. (Порушення цього правила вже коштувало
  переробки — не повторюй.)
- **Межі.** Один модуль не імпортує нутрощі іншого; ця підсистема самодостатня.
- **Залежності.** Додається рівно одна зовнішня бібліотека — `aiosmtplib`. Побудова листа —
  стандартною бібліотекою (`email.message.EmailMessage`), без додаткових пакетів.
- **Стек.** Python + FastAPI, керування пакетами `uv`, лінтер `ruff`. ОС розробки — Windows.
- **Стиль коду.** Канонічні рішення без `type: ignore` та обхідних прийомів.

## Обсяг (що входить)

1. **Залежність.** Додати `aiosmtplib` через `uv add aiosmtplib`.

2. **Конфігурація SMTP.** Розширити наявні застосункові налаштування (Pydantic settings,
   як для JWT) полями, що читаються з `.env`:
   - `SMTP_HOST: str`
   - `SMTP_PORT: int` (типово `587`)
   - `SMTP_USERNAME: str`
   - `SMTP_PASSWORD: str`
   - `SMTP_FROM: str` (адреса відправника у листі)
   - `SMTP_STARTTLS: bool` (типово `True`)

   Додати відповідні рядки до `.env.example` з поясненням Gmail/STARTTLS/app-password.
   Перенести ті самі змінні, що були в C#-беку (той самий провайдер, без стороннього API).

3. **Інтерфейс `EmailSender`.** Розмістити підсистему в `app/email/` (або в наявному
   аналозі каталогу спільної інфраструктури — узгодь із поточною структурою репозиторію
   перед створенням). Оголосити протокол:

   ```python
   class EmailSender(Protocol):
       async def send(
           self,
           *,
           to: str,
           subject: str,
           text: str,
           html: str | None = None,
       ) -> None: ...
   ```

   Це урізаний еквівалент C#-інтерфейсу `IEmailSender`: **без** вкладень, ретраїв і черги.
   Черга доставки свідомо відкладена — див. `DECISIONS.md`, запис D-002; не додавай її й не
   лишай для неї заглушок.

4. **Реалізація `SmtpEmailSender`.** Клас, що реалізує `EmailSender` поверх `aiosmtplib`:
   - будує `EmailMessage`: `From` = `SMTP_FROM`, `To` = `to`, `Subject` = `subject`,
     тіло — `text`; за наявності `html` додає альтернативну частину (`multipart/alternative`
     через `set_content` + `add_alternative(html, subtype="html")`);
   - надсилає через `aiosmtplib.send(...)` з параметрами `hostname`, `port`, `username`,
     `password`, `start_tls` з конфігурації;
   - конструктор приймає конфігурацію (значення полів вище), а не читає глобальні
     налаштування всередині методу.

5. **Провайдер залежності для FastAPI.** Функція-провайдер (`get_email_sender`), що повертає
   інстанс `SmtpEmailSender`, зібраний з налаштувань, — придатна для `Depends(...)`. Розмістити
   поряд з іншими провайдерами застосунку.

6. **Re-export.** `app/email/__init__.py` як чистий фасад: назовні видимі `EmailSender`,
   `SmtpEmailSender`, `get_email_sender`.

## Обсяг (що НЕ входить)

- Жодних вкладень, ретраїв, черги, таблиці `email_outbox`.
- Жодної логіки magic-link, жодних нових таблиць чи міграцій.
- Жодного реального мережевого з'єднання в тестах.

## Тести

Каталог тестів проєкту, стиль наявних тестів. Реальний SMTP не викликати —
підмінити `aiosmtplib.send` (patch на символі, який імпортує `SmtpEmailSender`).

Покрити:
1. **Текстовий лист.** Виклик `send(to=..., subject=..., text=...)` призводить до одного
   виклику транспорту; перевірити, що зібране повідомлення має коректні `From`/`To`/`Subject`
   і текстове тіло.
2. **HTML-альтернатива.** За переданого `html` повідомлення містить обидві частини
   (`text/plain` і `text/html`) як `multipart/alternative`.
3. **Параметри транспорту.** У виклик транспорту передано `hostname`, `port`, `username`,
   `password` та `start_tls` згідно з конфігурацією (зокрема `start_tls=True` за
   `SMTP_STARTTLS=True`).

## Критерії приймання

- `ruff` чисто (лінт і формат).
- Усі тести проходять; наявний набір не зламано.
- Жодного українського тексту в коді та коментарях.
- `EmailSender` — єдина точка, через яку решта застосунку надсилає пошту; місця виклику не
  знають про `aiosmtplib`.

## Після виконання

Повернути: перелік створених/змінених файлів, доданих `.env`-змінних і підсумок тестів
(кількість, статус). Не переходити до логіки magic-link — це окремий крок 2.
